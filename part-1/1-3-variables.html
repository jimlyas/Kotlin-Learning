1.3 Variables (val vs var)

Learn the difference between immutable (val) and mutable (var) variables, type inference, and best practices.

val - Immutable Variables

Use val for variables that won't change after initialization. This is similar to Java's final keyword. Prefer val whenever possible for safer, more predictable code.

Kotlin
fun main() {
    val name: String = "Kotlin"
    println(name)  // Kotlin
    
    val age = 25  // Type inferred as Int
    println(age)   // 25
}

Note: Once assigned, a val cannot be reassigned. However, if the value is an object, you can still modify its properties.

var - Mutable Variables

Use var for variables that need to change during the program's execution.

Kotlin
fun main() {
    var counter = 0
    println(counter)  // 0
    
    counter = 1
    println(counter)  // 1
    
    counter++
    println(counter)  // 2
}

Type Inference

Kotlin can automatically infer the type based on the assigned value. You don't always need to explicitly declare types.

Kotlin
fun main() {
    val message = "Hello"     // Inferred: String
    val count = 42           // Inferred: Int
    val pi = 3.14159        // Inferred: Double
    val isValid = true       // Inferred: Boolean
    
    println("$message $count $pi $isValid")
}

You can still declare types explicitly when needed for clarity or when the type cannot be inferred.
