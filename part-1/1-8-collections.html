1.8 Collections

Master Kotlin's collection types: List, Set, and Map, including both immutable and mutable variants.

List

A List is an ordered collection that allows duplicate elements. Kotlin distinguishes between read-only and mutable lists.

Kotlin
fun main() {
    val numbers = listOf(1, 2, 3, 4, 5)
    println(numbers[0])      // 1
    println(numbers.first())  // 1
    println(numbers.last())   // 5
    println(numbers.size)     // 5
    
    val mutable = mutableListOf(1, 2, 3)
    mutable.add(4)
    mutable.removeAt(0)
    println(mutable)  // [2, 3, 4]
}

Set

A Set is a collection of unique elements. It doesn't allow duplicates.

Kotlin
fun main() {
    val fruits = setOf("apple", "banana", "apple", "orange")
    println(fruits)  // [apple, banana, orange]
    println("banana" in fruits)  // true
}

Map

A Map stores key-value pairs. Keys are unique, but values can repeat.

Kotlin
fun main() {
    val ages = mapOf("Alice" to 30, "Bob" to 25)
    println(ages["Alice"])     // 30
    println(ages.keys)         // [Alice, Bob]
    println(ages.values)       // [30, 25]
    
    val mutable = mutableMapOf("x" to 1)
    mutable["y"] = 2
    println(mutable)  // {x=1, y=2}
}
