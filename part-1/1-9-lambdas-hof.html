1.9 Lambdas & Higher-Order Functions

Learn about anonymous functions, lambda expressions, and how functions can accept other functions as parameters.

Lambda Expressions

A lambda is a function without a name, defined inline. The last expression in a lambda is its return value.

Kotlin
fun main() {
    val sum: (Int, Int) -> Int = { a, b -> a + b }
    println(sum(3, 4))  // 7
    
    val numbers = listOf(1, 2, 3, 4, 5)
    val evens = numbers.filter { it % 2 == 0 }
    println(evens)  // [2, 4]
}

Higher-Order Functions

A higher-order function is one that takes functions as parameters or returns a function.

Kotlin
fun operation(a: Int, b: Int, op: (Int, Int) -> Int): Int {
    return op(a, b)
}

fun main() {
    println(operation(10, 5) { a, b -> a + b })  // 15
    println(operation(10, 5) { a, b -> a - b })  // 5
    println(operation(10, 5) { a, b -> a * b })  // 50
}

Common Collection Functions

Kotlin provides powerful functions like map, filter, and reduce for functional programming.

Kotlin
fun main() {
    val numbers = listOf(1, 2, 3, 4, 5)
    
    val doubled = numbers.map { it * 2 }
    println(doubled)  // [2, 4, 6, 8, 10]
    
    val sum = numbers.reduce { acc, n -> acc + n }
    println(sum)  // 15
    
    val squared = numbers.map { it * it }.filter { it > 10 }
    println(squared)  // [16, 25]
}
