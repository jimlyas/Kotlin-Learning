3.4 Dynamic Programming

Learn about DP optimization technique that solves complex problems by breaking them into simpler subproblems.

DP Concepts

Dynamic Programming (DP) optimizes recursive algorithms by storing previously computed results. It's applicable when a problem has:

• Optimal Substructure: Solution can be built from subproblem solutions

• Overlapping Subproblems: Same subproblems are computed multiple times

Climbing Stairs - Memoization vs Tabulation

Kotlin
// Memoization (Top-down) - O(n) time, O(n) space
fun climbStairsMemo(n: Int, memo: IntArray = IntArray(n + 1)): Int {
    if (n <= 2) return n
    if (memo[n] != 0) return memo[n]
    memo[n] = climbStairsMemo(n - 1, memo) + climbStairsMemo(n - 2, memo)
    return memo[n]
}

// Tabulation (Bottom-up) - O(n) time, O(1) space
fun climbStairsTab(n: Int): Int {
    if (n <= 2) return n
    var a = 1, b = 2
    for (i in 3..n) {
        val temp = a + b
        a = b
        b = temp
    }
    return b
}

LeetCode: Climbing Stairs

Easy

You are climbing a staircase. It takes n steps to reach the top. Each time you can either climb 1 or 2 steps. In how many distinct ways can you climb to the top?

View on LeetCode

This is essentially a Fibonacci problem. To reach step n, you can come from step n-1 (1 step) or step n-2 (2 steps). So ways(n) = ways(n-1) + ways(n-2).

Kotlin Solution
fun climbStairs(n: Int): Int {
    if (n <= 2) return n
    var a = 1, b = 2
    for (i in 3..n) {
        val temp = a + b
        a = b
        b = temp
    }
    return b
}
