2.1 Big O Notation

Learn how to analyze algorithm efficiency using Big O notation to measure time and space complexity.

What is Big O?

Big O notation describes the upper bound of an algorithm's growth rate. It tells us how an algorithm's performance changes as the input size increases.

When we say an algorithm is O(n), we mean the time complexity grows linearly with the input size.

Common Complexities

O(1) - Constant

Execution time doesn't change with input size. Array index access.

O(log n) - Logarithmic

Time doubles when input quadruples. Binary search.

O(n) - Linear

Time grows proportionally with input. Single loop.

O(n log n) - Linearithmic

Best sorting algorithms (merge sort, quick sort).

O(n²) - Quadratic

Time grows with square of input. Nested loops.

O(2^n) - Exponential

Doubles with each input. Fibonacci recursive.

Examples in Kotlin

Kotlin
// O(1) - Constant time
fun getFirst(arr: List<Int>): Int = arr[0]

// O(n) - Linear time
fun findMax(arr: List<Int>): Int {
    var max = arr[0]
    for (x in arr) {
        if (x > max) max = x
    }
    return max
}

// O(n²) - Quadratic time
fun printPairs(arr: List<Int>) {
    for (i in arr) {
        for (j in arr) {
            println("$i, $j")
        }
    }
}

Space Complexity

Space complexity measures how much memory an algorithm uses relative to input size.

When you create a new array of size n, that's O(n) space. When you use only a few variables, that's O(1) space.
